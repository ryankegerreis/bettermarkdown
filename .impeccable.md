## Design Context

### Users

BetterMarkdown is for anyone editing Markdown files directly, with the archetypal user being a desktop writer who wants formatting help without losing the clarity and portability of plain text.

### Brand Personality

Literary, quiet, precise. The interface should support concentration and feel like a well-made writing tool rather than a document-management system.

### Aesthetic Direction

Use the application’s existing native-feeling light and dark themes. Favor restrained editorial hierarchy, compact controls, and calm surfaces that recede behind the document. Avoid decorative effects, visual noise, and formatting controls that compete with the writing canvas.

### Design Principles

- Keep the document visually dominant and make secondary tools easy to dismiss.
- Use familiar writing and Markdown language instead of implementation jargon.
- Make formatting actions fast, predictable, and reversible.
- Preserve editor focus and keyboard-first workflows.
- Match the existing application chrome before adding new visual vocabulary.
